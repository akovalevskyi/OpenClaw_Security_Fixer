# OpenClaw Security Checklist (VPS / Host Level)

Этот документ описывает проверки, которые должны проводиться на уровне хоста (VPS), за пределами Docker-контейнера OpenClaw.

## 1. Настройка Firewall (UFW)
**Цель:** Изоляция сервера от открытого интернета.
- [ ] UFW установлен и активен (`sudo ufw status`).
- [ ] Правило по умолчанию: `deny incoming`, `allow outgoing`.
- [ ] SSH порт изменен со стандартного 22 на кастомный (например, 2244) и разрешен в UFW (`sudo ufw allow 2244/tcp`).
- [ ] Порты Docker/OpenClaw скрыты от внешнего мира, если не используется прокси.
- **Критичность:** Высокая. Оставленные открытыми порты баз данных или Docker API могут привести к взлому за секунды.

## 2. Защита от Брутфорса (Fail2ban)
**Цель:** Блокировка автоматизированных атак по перебору паролей.
- [ ] Fail2ban установлен и запущен (`systemctl status fail2ban`).
- [ ] Настроен `jail.local` для защиты SSH (особенно на кастомном порту).
- [ ] Настроены правила банов (например, бан на 10 минут после 3-х неудачных попыток).
- **Критичность:** Высокая. Боты постоянно сканируют IP-адреса; Fail2ban предотвращает истощение ресурсов сервера и снижает риск успешного брутфорса.

## 3. Защищенное Управление Секретами (Vault.sh)
**Цель:** Исключение хранения API ключей в конфигурациях и логах.
- [ ] Скрипт `vault.sh` настроен и имеет права `700` только для владельца.
- [ ] `vault_bridge.py` настроен для работы с Signal агентом и требует интерактивного ввода пароля.
- [ ] В `openclaw.json` и логах отсутствуют хардкод ключи (проверено скриптом `security_audit.py`).
- **Критичность:** Критическая. Токены OpenAI и Groq стоят денег. Если злоумышленник получит доступ к конфигурации, он украдет эти токены.

## 4. Изоляция Docker Контейнеров
**Цель:** Предотвращение побега из контейнера (Container Escape).
- [ ] Контейнер OpenClaw работает без флага `--privileged`.
- [ ] В `docker-compose.yml` добавлена опция `security_opt: ["no-new-privileges:true"]`.
- [ ] Директория `/data` примонтирована с правильными правами (на хосте `700` или `755` только для нужного пользователя).
- **Критичность:** Критическая. Побег из контейнера дает рут-права на хост-машине (VPS).

## 5. Регулярные Обновления (Maintenance)
**Цель:** Закрытие известных уязвимостей (CVE).
- [ ] Операционная система регулярно обновляется (`apt update && apt upgrade`).
- [ ] Образы Docker и npm пакет `openclaw` обновляются не реже раза в месяц.
- [ ] После каждого обновления запускается `security_audit.py` для выявления сброшенных настроек.
- **Критичность:** Высокая. Устаревшее ПО - главная причина взломов.
